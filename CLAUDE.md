# CLAUDE.md — 작업 원칙 및 프로세스

## 이 문서의 역할
이 문서는 구현자(Claude Code)의 작업 원칙과 프로세스를 정의한다.
SPEC.md가 "무엇을 만들 것인가"라면, 이 문서는 "어떻게 만들 것인가"이다.

---

## 핵심 원칙

### 1. 경험을 만들어라. 요소를 나열하지 마라.
CSS 속성을 넣고 체크하는 것이 아니라, 브라우저에서 열었을 때 느낌이 전달되는지가 기준.

### 2. Cosmos(cosmos.so)가 UX 레퍼런스다.
텍스트 설명을 해석하지 마라. `_refs/references.md`에 레퍼런스 자료가 있다.
스크롤 체감, 섹션 전환, 텍스트 등장/퇴장 방식에서 Cosmos를 기준으로 삼는다. 
특히 섹션 5(목업 데모)의 배치 패턴은 Cosmos 섹션 3 "Image by image"를 참고하라.

### 3. _refs/samantha.svg를 반드시 열어보라.
이 SVG가 디바이스 목업의 시각적 기준. 앱 실행 시퀀스(홈 화면 → 아이콘 확장 → 파동 구동)를 포함한다. 코드를 열어서 구조, 색상, 애니메이션 값을 직접 확인하라.
※ samantha.svg(v1)는 폐기됨. samantha_02.svg를 사용.

### 4. 안전한 쪽으로 수렴하지 마라.
언제나 **시각적 야망을 가지고 실행하라**. 과하면 줄일 수 있지만, 밋밋하면 살릴 수 없다.

### 5. 디자인 재량을 적극적으로 행사하라.
SPEC에 명시되지 않은 시각적 디테일(그림자 깊이, 그라데이션 미세 조정, 여백 비율, 모션 이징, 스니펫 배치 좌표)은 구현자가 판단하여 최상의 결과를 만들어라.

## 금지 사항

1. **순수 단색 배경 금지**. 모든 배경에 최소한의 그라데이션/텍스처/깊이감이 있어야 한다.
2. **모든 모션이 fade-in + translateY인 구현 금지**. 섹션별로 다른 모션 언어를 써라.
3. **760px 이하 좁은 컨테이너 금지**. 풀스크린 섹션 기반.
4. **AI 슬롭(generic AI aesthetics) 금지**. Inter, Roboto, Arial 사용 금지. 보라 그라데이션+흰 배경 금지. 예측 가능한 레이아웃 금지.
5. **요청과 무관한 기존 코드·주석·포매팅 수정 금지**. 건드릴 이유가 있으면 먼저 말해라.
6. **단일 용도 코드에 추상화·래퍼·설정 레이어 추가 금지**. 50줄로 될 것을 200줄로 만들지 마라.

## 기술 스택

- 단일 index.html (CSS + JS 인라인). 빌드 시스템 없음.
- GSAP + ScrollTrigger (CDN)
- Lenis v1.3.17+ (CDN) — wrapper 모드. cosmos.so 동일 아키텍처.
- Google Fonts / CDN: Noto Serif KR 300, Pretendard Variable.
- SVG 애니메이션: 파동은 SVG animate + CSS @keyframes. GSAP은 스크롤 연동에만.
- Web3Forms: 폼 백엔드. AJAX 제출. botcheck honeypot.
